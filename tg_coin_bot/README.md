# tg_coin_bot — запуск и остановка

## Виртуальное окружение (venv)

Чтобы зависимости бота не смешивались с системным Python, удобно использовать **venv**.

**Создать** (один раз), из каталога `tg_coin_bot`:

```bash
cd tg_coin_bot
python3 -m venv .venv
```

**Активировать** окружение — в том же терминале перед `pip` и `python main.py`:

- **macOS / Linux** (zsh, bash):

  ```bash
  source .venv/bin/activate
  ```

- **Windows** (cmd / PowerShell):

  ```cmd
  .venv\Scripts\activate
  ```

В приглашении строки обычно появится префикс `(.venv)` — значит, `python` и `pip` идут из этого окружения.

**Деактивировать** (выйти из venv, вернуться к обычному Python в этом терминале):

```bash
deactivate
```

Команда `deactivate` **не останавливает** бота. Сначала останови процесс бота (`Ctrl+C` в окне, где запущен `python main.py`), потом при желании выполни `deactivate`.

## Активировать бота (запустить)

1. **Python и зависимости**  
   При использовании venv сначала `source .venv/bin/activate` (см. выше). Затем из каталога `tg_coin_bot`:

   ```bash
   pip install -r requirements.txt
   ```

2. **Переменные окружения**  
   Скопируй `.env.example` в `.env` в каталоге `tg_coin_bot` и заполни как минимум:
   - `BOT_TOKEN` — токен от [@BotFather](https://t.me/BotFather)
   - `ADMIN_IDS` — Telegram user id админов через запятую (для админ-команд)

   Опционально: `DB_PATH` (по умолчанию `bot.db` для SQLite), `MYSQL_URL` / `DATABASE_URL` для MySQL на проде — см. комментарии в `.env.example` и сообщение об ошибке в `main.py`, если токен не задан.

3. **Запуск из каталога бота**

   ```bash
   cd tg_coin_bot
   python main.py
   ```

   В консоли должно появиться что-то вроде `Bot started: @your_bot ...` — после этого бот слушает обновления через **long polling**.

4. **Webhook**  
   При старте бот вызывает `delete_webhook`, чтобы polling получал апдейты. Если вы снова включите webhook у этого же бота в другом месте, этот процесс перестанет получать сообщения, пока webhook не снимут.

## Деактивировать бота (остановить)

- **Локально в терминале:** нажми `Ctrl+C` в сессии, где запущен `python main.py` (это останавливает **процесс бота**, не venv).
- **Фоновый процесс:** найди PID и заверши процесс (`kill <pid>`), либо останови сервис, если бот запущен как systemd / Docker / процесс на Railway — там используйте штатную кнопку «Stop» или перезапуск без старта контейнера.

После остановки процесса бот в Telegram перестаёт отвечать, пока вы снова не запустите `main.py` (или не поднимете другой инстанс с тем же токеном — двух активных polling на один токен быть не должно).

## Дополнительно: «активные» активности в базе

В таблице `activities` есть поле **`is_active`** (`1` — включена, `0` — выключена). Неактивные активности не участвуют в начислениях через логику бота (запросы идут с условием `is_active = 1`).

При каждом **инициализировании схемы БД** при старте бота список активностей из кода (`_activities_seed_rows()` в `main.py`) синхронизируется с базой: строки из этого списка получают `is_active = 1`, все остальные по `title` — **`is_active = 0`**.

Чтобы **вручную** временно отключить одну активность, не меняя код, можно выполнить в SQLite / MySQL:

```sql
UPDATE activities SET is_active = 0 WHERE title = 'Точное название активности';
```

Имя должно совпадать с тем, что в базе (как в сиде в `main.py`). Учтите: при следующем запуске бота сид снова может выставить `is_active = 1` для строк из списка в коде и обнулить остальные — для постоянной схемы лучше править список в `_activities_seed_rows()` или не полагаться только на ручной `UPDATE`, если вам нужна стабильная «выключенная» активность из сида.
